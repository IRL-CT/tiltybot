# TiltyBot ワークショップガイド

TinyPICO、Dynamixelモーター、オープンソースソフトウェアを使って、スマートフォンで操作できるロボットを作るハンズオンワークショップです。

## 1. 事前準備

ワークショップ開始前に行う準備です。どこかのステップができなくても心配ありません。当日、対面でサポートいたします。

### 始める前の確認

- **Macをお使いの方：** OSのバージョンが **macOS Sonoma以降** であることを確認してください。
- **iPhoneをお使いの方：** iOSのバージョンが **17.3.1以降** であることを確認してください。
- ロボットの操作には **個人のスマートフォン** を使用します。企業管理のスマートフォンはセキュリティソフトウェアの制限により使用できない場合があります。

### ステップ1：Visual Studio CodeとGitのインストール

以下のリンクからダウンロードし、デフォルト設定のままインストールしてください。

- [Visual Studio Code](https://code.visualstudio.com/)
- [Git](https://git-scm.com/download/win)（Windows用）/ Macには通常プリインストールされています

### ステップ2：TiltyBotリポジトリのクローン

プロジェクトのリポジトリ（ソースコードやファイルが入ったフォルダ）をパソコンの任意の場所（デスクトップなど）にクローンします。

1. ターミナルを開きます（Windowsの場合は保存先のフォルダで右クリック →「**Git Bash Here**」を選択）。

![フォルダからGit Bashを開く](img/GitBashHere.png)

2. 以下のコマンドを実行します：

```
git clone https://github.com/imandel/tiltybot.git
```

![リポジトリのクローン](img/GitClone.png)

### ステップ3：PlatformIOのインストール（Visual Studio Code上）

1. VS Codeで **View → Extensions** を開きます。**PlatformIO** を検索してインストールします。

![PlatformIO拡張機能のインストール](img/image11.png)

2. 左サイドバーのPlatformIOアイコン（アリのマーク）![アリのアイコン](img/image5.png)をクリックして初期設定を開始します。右下に「Finished」と表示されるまで待ちます。

![PlatformIO初期設定中](img/image14.png)

3. 「**Reload Now**」と表示されたらクリックしてリロードします。

### ステップ4：プロジェクトのビルド

1. PlatformIOでTiltyBotリポジトリを開きます：アリのアイコン → **Quick Access → Open** → クローンした `tiltybot` フォルダを選択。

![PlatformIOでプロジェクトを開く](img/image12.png)

2. 左端のファイルエクスプローラーから `platformio.ini` を開きます。

![platformio.iniの選択](img/image10.png)

3. 画面下部の **ビルドボタン**（チェックマーク）を押してビルドを実行します。

![ビルドボタンの位置](img/image15.png)

以上で事前準備は完了です！

## 2. ソフトウェアセットアップ — ベースロボットの作り方

ベースロボットには3つのコントロールモードがあります：

- **ドライブモード：** 車輪を使って動き回るモバイルロボット。
- **ティルティモード：** 顔を傾けたり見回したりできるダッシュボード型ロボット。
- **2モーターモード：** 2つのモーターを0〜360°で直接制御。オリジナルロボットの設計に便利。

### 始める前に

- OS/iOSバージョンが要件を満たしていることを確認してください（事前準備セクション参照）。
- TinyPICOとDynamixelモーターをUSBでパソコンに接続してください。

![TinyPICOとモーターをパソコンに接続](img/IMG_7822.jpg)

### ステップ1：コントロールモードの選択

VS Codeで `platformio.ini` を開きます。28行目付近に以下の3つのソースファイルがあります：

```
+<2motor.cpp>
+<tilty.cpp>
+<drive.cpp>
```

使用したいモードの行を有効にし、残りの2つを行頭に `;` を付けてコメントアウトします。例えば、2モーターモードを有効にする場合：

```
+<2motor.cpp>
;+<tilty.cpp>
;+<drive.cpp>
```

![platformio.iniでモードを選択](img/SelectingMode.png)

以降の手順では **2モーターモード** を使用します。他のモードに変更したい場合は、このファイルを編集して再アップロードしてください。

### ステップ2：Wi-Fi情報の設定

ロボットはローカルWi-Fiネットワークのホストになります（インターネット接続はありません）。スマートフォンでこのネットワークに接続し、ブラウザからロボットを操作します。

`network.h` を開き、SSIDとパスワードを設定してください。他の参加者と重複しないよう、自分の名前やチーム名など分かりやすい名前を推奨します。

![network.hの編集](img/networkh.png)

### ステップ3：ファームウェアのアップロード

画面下部の **アップロードボタン**（→矢印）を押して、C++で書かれたファームウェアをTinyPICOにアップロードします。このファームウェアは、Wi-Fiネットワークのホスティング、Webサーバの起動、ロボットとブラウザ間の通信を担当します。

![アップロードボタン](img/Upload.png)

「SUCCESS」と表示されれば完了です：

![アップロード成功](img/image7.png)

### ステップ4：ファイルシステムイメージのビルド

ブラウザ用のインターフェース（HTML、JavaScript、CSS）は別途フラッシュメモリにアップロードする必要があります。

**PlatformIOアリアイコン → Platform → Build Filesystem Image** をクリックします。

![Build Filesystem Imageメニュー](img/SerialMonitor.png)

「SUCCESS」と表示されれば完了です：

![ビルド成功](img/image7.png)

### ステップ5：ファイルシステムイメージのアップロード

**PlatformIOアリアイコン → Platform → Upload Filesystem Image** をクリックします（OTAではありません）。

![Upload Filesystem Imageメニュー](img/UploadfilesystemImage.png)

**重要：** アップロード前にシリアルモニターが閉じていることを確認してください。開いている場合は、下部のMonitorタブにカーソルを合わせてゴミ箱アイコンをクリックして閉じてからアップロードしてください。

![アップロード前にシリアルモニターを閉じる](img/image9.png)

### ステップ6：スマートフォンとロボットの接続

1. VS Code下部の **シリアルモニター**（コンセントのアイコン）を開き、IPアドレスを確認します。表示されない場合はTinyPICOの **リセットボタン** を押してください。

![シリアルモニターにIPアドレスが表示](img/image18.png)

リセットボタンはデバイスの表面にあります：

![リセットボタンの位置](img/IMG_7790.jpg)

2. スマートフォンのWi-Fi設定で、ステップ2で設定したネットワークに接続します。「インターネット接続なし」等の警告は無視してください。プライバシーに関する警告も表示される場合がありますが、そのまま接続してください。

![スマートフォンのWi-Fiと証明書の設定](img/SettingForPhone.png)

3. スマートフォンのブラウザで、シリアルモニターに表示されたURL（例：`https://192.168.X.X/2motor.html`）にアクセスします。

4. コントロールパネルの **Activate** を押し、画面上のスライダーを操作します。シリアルモニターに数値が表示されれば接続成功です。

![2モーターモードのスマートフォン操作画面](img/IMG_7806.PNG)

![シリアルモニターにモーター値が表示](img/2motermodeZero.png)

**ヒント：** スマートフォンのモバイルデータ通信をOFFにすると、通信が自動的に切り替わるのを防げます。VPNに接続している場合は切断してからアクセスしてください。

**重要：** モーターを取り外す前に、両方のモーターのポジションを必ず **0** に設定してください。手でモーターの位置を無理に変えないでください。ティルティロボットの組立て時に特に重要です。

### ステップ7：オーディオ機能

ロボットはBluetoothスピーカーを使って音を出すこともできます。

**Bluetoothスピーカーの接続：**

1. Bluetoothスピーカーの電源を入れます。

![Bluetoothスピーカーの電源ボタン](img/IMG_7758.png)

2. スマートフォンのBluetooth設定からスピーカーに接続します。

![スマートフォンのBluetoothペアリング](img/IMG_8803.PNG)

**録音と再生：**

1. スマートフォンのブラウザで `https://{IPアドレス}/sounds.html` にアクセスします。マイクへのアクセス許可を求められた場合は許可してください。

![サウンドページとマイク許可](img/IMG_8798.PNG)
![マイクアクセスの許可プロンプト](img/IMG_8797.PNG)

2. **Record** ボタンを押して録音を開始します。タイマーで録音時間を確認できます。**Stop** ボタンを押して録音を終了します。

![録音中](img/IMG_8799.PNG)

3. ポップアップで任意の名前を入力し「OK」を押して保存します。

4. **Play** ボタンを押して録音を再生します。

![再生コントロール](img/IMG_8800.PNG)

**注意：** コントロール画面をリロードすると録音したデータが消えてしまいます。

## 3. ハードウェアの組立て

以下の2種類のロボットから選んで組み立てます：

| ドライブロボット | ティルティロボット |
|---|---|
| 車輪で移動するモバイルロボット | 傾き・パン動作ができるダッシュボード型ロボット |

### ベースの組立て

コア部品（TinyPICO＋Dynamixelモーター）を下の写真のように組み立て、USB A-Cケーブルでパソコンに接続します。

![ベースロボットの組立て](img/IMG_7787.JPG)

### ドライブロボットの組立て

1. **軸の取り付け：** ネジを使って白い軸パーツを各モーターに取り付けます。

![モーターに軸パーツを取り付け](img/IMG_7785.jpg)
![軸用のネジ](img/screws_axel.jpg)

2. **タイヤの取り付け：** 軸をタイヤに差し込みます。

![タイヤを軸に取り付け](img/IMG_7817.jpg)

3. **台の作成：** ダンボールなどの素材を使って、モーターを取り付ける台を作ります。

![ダンボールの台](img/IMG_7818.jpg)

4. **部品の配置：** 2つのDynamixelモーターと360度タイヤを台に取り付けます。三角形の配置になるようバランスを取ってください。

![モーターとローラーボールの配置](img/IMG_7819.jpg)
![完成したドライブロボットのベース](img/IMG_7844.jpg)

5. **ドライブモードのアップロード：** `platformio.ini` を編集して `+<drive.cpp>` を有効にし、他をコメントアウトします。ファームウェアをアップロードします。

![ドライブモードのplatformio.iniとアップロード](img/drivemode_upload.png)

**注意：** モードによってアクセスするURLのパスが変わります（`/drive.html`、`/2motor.html`、`/tilty.html`）：

![モードごとのURL変更](img/URL.png)

6. **スマートフォンの接続：** ロボットのWi-Fiネットワークに接続し、シリアルモニターに表示されたURL（例：`https://192.168.X.X/drive.html`）にアクセスします。

![ドライブモードのIPアドレス表示](img/Drivemode_IPaddress.png)

7. **操作：** Activateを押して、画面上のジョイスティックでロボットを操作します。

![ドライブモードの動作](img/drive_mode2.gif)

**前提条件：** 組み立て前に2モーターモードで両方のモーターをポジション0に設定してください。

1. **モーターをゼロに設定：** 2モーターモードのコントロールページに接続し、両方のモーターポジションを0にします。

![スマートフォンでモーターをゼロに設定](img/MoterZeroScreenshot.png)
![シリアルモニターでゼロ確認](img/2motermodeZero.png)

2. **ヘッドの取り付け：** 短いネジを使って、ヒンジブラケットを **モーター1** に取り付けます。ポジション0の状態で「ヘッド」が前方を向くようにします。

![モーター1にヘッドを取り付け](img/IMG_7811.JPG)
![ブラケット用のネジ](img/screws_blaket.jpg)

3. **アイドラーの取り付け：** **モーター2** にアイドラーとアイドラーキャップのみを取り付けます。

![アイドラー用のネジ](img/screws_idler.jpg)

4. **組み合わせ：** ヘッド/ヒンジアセンブリをモーター2に正しい向きで取り付けます（モーター2上部のLEDが手前を向くようにします）。**ホイールを手で回転させないでください！**

![正しい組立て向き](img/IMG_7789.jpg)

5. **ティルティモードのアップロード：** `platformio.ini` を編集して `+<tilty.cpp>` を有効にし、他をコメントアウトします。ファームウェアをアップロードします。

![ティルティモードのアップロード](img/Tilty_upload.png)

6. **スマートフォンの接続：** Wi-Fiネットワークに接続し、シリアルモニターに表示されたURLにアクセスします。

7. **操作：** Activateを押して、画面上のコントロールまたはスマートフォンのジャイロスコープでロボットを傾けたりパンしたりできます。

![ティルティモードの動作](img/PanTilt_1.gif)

ロボットが完成したら、人とロボットのインタラクション（やり取り）について考えます。このセクションでは、デザイン思考のエクササイズを行います。

A. [プランニング](#part-a-プランニング)
B. [インタラクションを演じる](#part-b-インタラクションを演じる)
C. [見た目をデザインする](#part-c-見た目をデザインする)

### Part A: プランニング

ロボットと人のインタラクションを設計するために、以下の4つの要素を考えてみてください。

1. **設定（Setting）：** そのインタラクションはどこで、いつ起こりますか？（例：キッチン、病院、屋外など）
2. **プレイヤー（Players）：** 誰が関わりますか？メインのユーザーだけでなく、周囲にいる人全員を考えてください。悪いデザインの例として、Amazon Alexaがルームメイトや子供の存在を十分に考慮せず設計されたことで、意図しない注文が行われるという問題が起きたケースがあります。
3. **アクティビティ（Activity）：** プレイヤーとロボットの間で何が起こりますか？
4. **ゴール（Goals）：** 各プレイヤーの目標は何ですか？（例：通知を受け取る、空間を移動する、感情を表現するなど）

設定・プレイヤー・アクティビティ・ゴールを書き出してください。

### Part B: インタラクションを演じる

プランしたシナリオをロボットなしで実際に演じてみてください。デバイスの役割は人が代わりに演じます。

演じた後に振り返りましょう：

- 紙の上では良さそうだったのに、実際にはうまくいかなかったことはありますか？
- 演じてみて新しいアイデアや発見はありましたか？

### Part C: 見た目をデザインする

次に、ロボットの見た目を考えます。以下の点を検討してください：

- 環境：オーバーヒートの可能性は？水濡れのリスクは？緊急時に目立つ色が必要？
- ユーザー：どのような形状やデザインが、使う人にとって自然で親しみやすいですか？

紙にスケッチを描き、ダンボールやテープなどの素材を使って実際にプロトタイプを作ってみましょう。

### Part D: デバイスを操作する

シナリオ通りにロボットを動かす練習をします。一人がコントローラーを操作し、もう一人がユーザーとしてロボットとやり取りします。

### Part E: 撮影する

プロトタイプのインタラクションを動画に撮影してください。設定、プレイヤー、ロボットの動きを含むシナリオ全体を記録します。この動画はアイデアのプレゼンテーションやさらなる改善に活用できます。
